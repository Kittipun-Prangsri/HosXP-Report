# ระบบขอรายงาน HOSxP (Version 2)

ระบบขอรายงานข้อมูลจากระบบ HOSxP โรงพยาบาลคลองหาด พัฒนาด้วย Google Apps Script ร่วมกับ HTML5, Vanilla CSS และ JavaScript ในการบริหารจัดการคิวการขอรายงานสำหรับหน่วยงานและบุคลากรภายในโรงพยาบาล พร้อมมีระบบแจ้งเตือนอัจฉริยะผ่านทาง LINE และ Telegram

## 🎨 การปรับแต่งเลย์เอาต์เพื่อการรองรับการใช้งานบนมือถือ (Mobile UX Optimization)

เพื่อมอบประสบการณ์การใช้งานบนสมาร์ทโฟนและอุปกรณ์แท็บเล็ตให้ยอดเยี่ยมเสมือนเป็นแอปพลิเคชันมือถือแท้ (Native App-Like Experience) ระบบได้ติดตั้งและจัดการองค์ความรู้การจัดทำหน้าจอแบบตอบสนอง ดังนี้:

### 1. การกำหนดขนาดขอบเขตภาพ (Apps Script Viewport Configuration)
เนื่องจาก Google Apps Script จะแสดงผลหน้าเว็บแอปพลิเคชันอยู่ภายในเฟรม `<iframe>` ส่งผลให้หน้าจอโมบายถูกย่อขนาดลงโดยอัตโนมัติหากเข้าจากเว็บตรง ระบบจึงกำหนด Viewport ในไฟล์ backend `รหัส.js` ผ่านคำสั่ง:
```javascript
HtmlService.createTemplateFromFile("Index")
  .evaluate()
  .addMetaTag("viewport", "width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no")
```
ทำให้การแสดงผลบนขนาดหน้าจอโทรศัพท์สอดคล้องกับขนาดสเกลจริงในสัดส่วน 1:1

### 2. การแปลงตารางข้อมูลเป็นระบบการ์ด (Table-to-Card Layout)
เนื่องจากหน้าจอตารางคิวคำขอและตารางรายชื่อผู้ใช้มีรายละเอียดที่ยาวเกินกว่าพื้นที่ความกว้างหน้าจอมือถือ ระบบจึงแปลงโครงสร้างแถวและคอลัมน์ดั้งเดิมของตารางให้แสดงผลแยกออกมาเป็นการ์ดอิสระในจอมือถือ:
* **เทคนิค**: ซ่อนส่วนหัวตาราง (`<thead>`) และแปลง `<tr>` / `<td>` ให้เป็นบล็อกแสดงผลเดี่ยว (`display: block`) 
* **การใส่ฉลากกำกับ**: ใช้ CSS Pseudo-elements `::before` ในการแทรกฉลากชื่อข้อมูลไว้ทางซ้าย (เช่น "เลขที่คำขอ:", "สถานะ:") และมีค่าจริงอยู่ด้านขวาเพื่อให้อ่านง่าย
```css
@media (max-width: 768px) {
  table, tbody, tr, td { display: block !important; width: 100% !important; }
  thead { display: none !important; }
  tbody td:nth-child(1)::before { content: "เลขที่คำขอ:"; font-weight: bold; }
}
```

### 3. กล่องข้อมูลแบบเต็มหน้าจอ (Full-Screen Modals)
เมื่อผู้ใช้งานเรียกดูรายละเอียดคำขอหรือกดปุ่มแก้ไขข้อมูลบนโทรศัพท์มือถือ กล่องโต้ตอบแบบมีเงาเดิมจะทำการกางแผ่ออกเต็มหน้าจอสมาร์ทโฟน (Full-screen Overlay) และมีจุดควบคุมการบันทึกหรือการปิดตรึงไว้ที่ท้ายหน้าจอ เพื่อให้ความรู้สึกที่เป็นระเบียบตามสไตล์แอปพลิเคชันมือถือ

### 4. ขนาดการสัมผัสและการป้อนข้อมูล touch-friendly (Touch Targets)
* **Touch Targets >= 52px**: ขยายขนาดความสูงขั้นต่ำของอินพุต ปุ่มกด และตัวเลือกในฟอร์มขึ้นเป็น 52px เพื่อให้อุปกรณ์ระบบจอสัมผัสสามารถกดเลือกได้ง่ายดาย
* **iOS WebKit Prevent Auto-Zoom**: ล็อกขนาดตัวอักษรขององค์ประกอบฟอร์มทุกตัวบนอุปกรณ์ขนาดเล็กไว้ที่ระดับ 17px เพื่อป้องกันไม่ให้เบราว์เซอร์ของระบบ iOS บนเครื่อง iPhone ซูมหน้าจอเข้าโดยอัตโนมัติในตอนที่ผู้ใช้แตะป้อนค่า

### 5. แดชบอร์ดประหยัดพื้นที่แนวตั้ง (Dashboard Compact Grid)
ปรับการ์ดกล่องแสดงผลแดชบอร์ดสถิติในหน้าจอมือถือจากที่เรียงซ้อนกันเป็นแถวเดี่ยวยาวลงไป ให้แสดงผลในแบบ **2 คอลัมน์ (2-Column Grid)** โดยให้การ์ด "คำขอทั้งหมด" มีขนาดแผ่เต็มความกว้างแถวแรก ทำให้ข้อมูลดูกระชับและไม่ต้องใช้นิ้วปัดหน้าจอลงไปลึกมากเพื่อตรวจสอบ

## 🛠️ คู่มือการตั้งค่าเครื่องมือและการเชื่อมต่อระบบ (Integrations & Tools Guide)

เพื่อให้ระบบทำงานได้อย่างเต็มประสิทธิภาพ รวมถึงการแจ้งเตือนเหตุการณ์สำคัญไปยังหน่วยงานไอทีและผู้ร้องขอโดยอัตโนมัติ ระบบนี้ใช้เทคโนโลยีหลักและเครื่องมือต่าง ๆ ดังต่อไปนี้:

### 1. clasp (Command Line Apps Script Projects)
* **ความสำคัญ**: เป็นเครื่องมือ CLI ของ Google ที่ทำให้นักพัฒนาสามารถพัฒนาเขียนโค้ดและจัดเรียงไฟล์ Google Apps Script จากคอมพิวเตอร์ของตนเอง (Local Environment) ได้ แทนการเขียนบนหน้าเว็บเบราว์เซอร์ของ Google Script
* **การใช้งานและการสั่งงาน**:
  - `clasp login` : สั่งล็อกอินเชื่อมต่อคอมพิวเตอร์ของคุณเข้ากับบัญชี Google
  - `clasp clone <scriptId>` : ดึงไฟล์โปรเจกต์จากระบบ Cloud มาไว้ที่ไดรฟ์เครื่องคอมพิวเตอร์
  - `clasp push` : ส่งโค้ดที่อัปเดตในเครื่องขึ้นไปยังเซิร์ฟเวอร์ Google App Script
  - `clasp push -w` : (Watch Mode) รันคำสั่งนี้ทิ้งไว้ เครื่องมือจะตรวจจับความเปลี่ยนแปลงเมื่อคุณกดบันทึกโค้ด (Save) และส่งขึ้น Cloud ทันทีโดยไม่ต้องพิมพ์สั่งใหม่บ่อย ๆ

### 2. LINE Messaging API (LINE Official Account)
ระบบใช้ LINE OA ในการส่งข้อความประเภท Flex Message เพื่อให้ข้อมูลมีดีไซน์ที่สวยงามและรองรับปุ่มกดลิงก์ต่าง ๆ ได้
* **วิธีกำหนดค่า Token**:
  1. เข้าสู่ระบบ [LINE Developers Console](https://developers.line.biz/)
  2. สร้าง Provider และจัดทำแชลเนลประเภท **Messaging API**
  3. ไปที่แท็บ **Messaging API** เลื่อนลงไปข้างล่างสุดตรงหัวข้อ **Channel access token (long-lived)** แล้วกดสร้าง (Issue) คัดลอกโทเคนมาใส่ที่ตัวแปร `LINE_ACCESS_TOKEN` ในไฟล์ `รหัส.js`
* **การระบุห้องแชทกลุ่มแอดมิน (Admin Group ID)**:
  1. นำบอท LINE OA ของคุณเชิญเข้าร่วมกลุ่มแชทไอทีที่ต้องการให้แจ้งเตือน
  2. นำลิงก์ Web App URL ไปใส่ในช่อง **Webhook URL** บน LINE Developers Console จากนั้นสับสวิตช์เปิดใช้ระบบ Webhook (**Use Webhook**)
  3. ให้ใครก็ได้ในกลุ่มพิมพ์ทักทายหรือพิมพ์ข้อความลงในแชทกลุ่มที่มีบอทอยู่
  4. ไปดูที่ Google Sheet ของคุณในหน้าแผ่นงาน **`Logs`** จะพบแถวที่มีคอลัมน์กิจกรรมแสดงข้อความ `GET_ID` โดยมีรายละเอียดกลุ่มปรากฏขึ้น (เช่น `C7d8e9a2b...` หรือ `Ud1ee9e2d...`)
  5. คัดลอก ID ของกลุ่มนั้นไปวางใส่ในค่าตัวแปร `LINE_ADMIN_GROUP_ID` ในไฟล์ `รหัส.js`
* **การใช้งานฝั่งบุคลากรทั่วไป (User LINE ID)**:
  - ผู้ใช้งานสามารถนำค่า **LINE User ID** ส่วนตัว (ซึ่งหาได้จากการทักแชทบอทตรง ๆ หรือเข้าลิงก์ตรวจสอบ ID) นำมาระบุและบันทึกในเมนู **"จัดการโปรไฟล์ส่วนตัว"** บนหน้าเว็บแอปได้ทันที เพื่อเปิดรับการแจ้งเตือนส่วนตัวส่งตรงไปยังโทรศัพท์มือถือเมื่อรายงานของตนเองดำเนินการเสร็จสิ้น

### 3. Telegram Bot API
ช่องทางแจ้งเตือนทางเลือกเสริม สำหรับกลุ่มแอดมินไอทีหรือผู้ใช้งานส่วนบุคคลที่ไม่ต้องการรับข่าวสารผ่านแอปพลิเคชัน LINE
* **ขั้นตอนการขอ Bot Token**:
  1. ค้นหาบอทต้นทางที่ชื่อ `@BotFather` ในแอปพลิเคชัน Telegram
  2. พิมพ์คำสั่ง `/newbot` และดำเนินการตั้งชื่อรวมถึงกำหนด Username บอทให้เรียบร้อย
  3. ระบบจะส่ง **HTTP API Token** กลับมาให้ (เช่น `7931324702:AAGVf...`) นำคีย์นี้ไประบุในตัวแปร `TELEGRAM_BOT_TOKEN` ในไฟล์ `รหัส.js`
* **การระบุ Chat ID สำหรับแจ้งเตือน**:
  - **สำหรับกลุ่มไอทีแอดมิน**: เชิญบอทของคุณเข้าสู่กลุ่ม Telegram แอดมิน จากนั้นเชิญบอทแสดงผลข้อมูลกลางอย่าง `@RawDataBot` เข้ากลุ่มชั่วคราวเพื่อตรวจสอบและแสดงหมายเลข ID ของกลุ่ม (ปกติ ID กลุ่มใน Telegram จะขึ้นต้นด้วยเครื่องหมายลบ `-` เช่น `-100223456789`) นำคีย์ตัวเลขนี้ไปใส่ในตัวแปร `TELEGRAM_ADMIN_CHAT_ID` ในไฟล์ `รหัส.js`
  - **สำหรับผู้ใช้งานทั่วไป**: ผู้ใช้คนใดที่ประสงค์ต้องการรับข้อความแจ้งเตือนทาง Telegram สามารถทักแชทไปหาบอทบิการ `@userinfobot` เพื่อเรียกดู Chat ID ส่วนตัวที่เป็นหมายเลขตัวเลข 9-10 หลัก จากนั้นนำคีย์นั้นมาคัดลอกลงในฟอร์ม **"จัดการโปรไฟล์ส่วนตัว"** บนหน้าเว็บแอป

### 4. โครงสร้างฐานข้อมูล Google Sheets (Sheet Database Structure)
ระบบใช้ตาราง Google Sheets ในการบันทึกข้อมูลหลัก โดยแยกแผ่นงาน (Tab) ออกเป็น 3 แผ่นงานหลักที่ต้องกำหนดชื่อและคอลัมน์ให้ตรง ดังนี้:

* **แผ่นงาน `Users` (ฐานข้อมูลผู้ใช้)**:
  `[A] username` | `[B] password` | `[C] displayName` | `[D] email` | `[E] role` | `[F] department` | `[G] lineId` | `[H] telegramId`
* **แผ่นงาน `Requests` (คิวแจ้งขอรายงาน)**:
  `[A] requestId` | `[B] username` | `[C] displayName` | `[D] department` | `[E] reportName` | `[F] purpose` | `[G] fieldsNeeded` | `[H] criteria` | `[I] requestDate` | `[J] status` | `[K] adminHandler` | `[L] downloadUrl` | `[M] adminNote`
* **แผ่นงาน `Logs` (ประวัติการใช้งานและตรวจสอบระบบ)**:
  `[A] timestamp` | `[B] level` | `[C] action` | `[D] message`

## 🚀 การติดตั้งและเผยแพร่ใช้งาน (Deployment)

1. ทำการลงทะเบียนใช้งานเครื่องมือ CLASP ของ Google เพื่อใช้งานกับเครื่องโลคอล
2. อัปโหลดไฟล์ขึ้นสู่ Cloud:
   ```bash
   clasp push
   ```
3. เปิดหน้าต่าง Google Apps Script ขึ้นมา -> ไปที่ **"การเปิดใช้งาน (Deploy)"** -> **"การทดสอบการใช้งานใหม่ (New Deployment)"**
4. เลือกประเภทการติดตั้งเป็น **"เว็บแอปพลิเคชัน (Web App)"**
5. ตั้งค่าการกำหนดสิทธิ์:
   * **ทำงานในฐานะ (Execute as)**: เลือกเป็นชื่อบัญชีเจ้าของระบบ (บัญชีของคุณ)
   * **ใครมีสิทธิ์เข้าถึง (Who has access)**: เลือกเป็นทุกคน (Anyone) หรือตามแต่ขอบเขตความปลอดภัยขององค์กร
6. กดปุ่ม **Deploy** เพื่อรับลิงก์ URL สำหรับเปิดเข้าสู่หน้าหลักของระบบขอรายงานข้อมูล
